How I think

The principles that guide every build, every line of code, every pixel placed.

01

Feeling First

Every project starts with a feeling, not a wireframe. What should the user experience? What emotion should they carry?

02

Code as Craft

Clean code isn't just functional — it's beautiful. Architecture matters. Naming matters. The details nobody sees are the ones that matter most.

03

AI as Collaborator

I don't compete with AI. I dance with it. The best work comes from the intersection of human intuition and machine capability.

04

Never Generic

If it looks like everything else, it's wrong. Every project needs a point of view, a personality, a reason to exist.

The process

Step 01

Listen & Absorb

Every project begins with deep understanding. What's the context? Who's the audience? What's the vibe? I immerse myself in the problem before touching a single line of code.

Step 02

Dream in Wireframes

Mental prototyping. I sketch layouts in my head, explore color palettes, test typography combinations. The best designs are born from iteration before implementation.

Step 03

Vibe Code

AI-assisted, human-directed. I use tools to accelerate, but the creative decisions are always mine. The code flows when the vision is clear.

Step 04

Refine Until It Hurts

The last 10% takes 90% of the time. Spacing, timing, easing, contrast. I polish until every interaction feels inevitable.
